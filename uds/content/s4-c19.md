## bar
:::en
To change what an ECU will let you do, you ask it to switch **session**. The tester sends `10 <session>`; the ECU checks the request, **flips its internal mode**, then replies `50 <session> …` with the timing limits for that mode. One request, one response — but a real state change happens inside in between.
:::
:::jp
ECUに「できること」を変えてもらうには、**セッション**の切り替えを頼みます。テスターが `10 <session>` を送ると、ECUは要求を確認し、**内部モードを切り替え**、そのモードのタイミング上限とともに `50 <session> …` を返します。往復は1回でも、その間に本当の状態変化が起きています。
:::

## leg:inside
:::en
The switch isn't instant magic — the **DCM** (the ECU's diagnostic handler) runs a short internal sequence before it answers, and can still **refuse**.
:::
:::jp
**DCM**（ECUの診断担当）は、応答する前に短い内部手順を踏み、**拒否**することもあります。
:::
:::figure src=figures/c19-dcm-sequence.svg
en: **validate → flip the session → arm timers → confirm.** A failed check returns `7F` and the mode never changes.
jp: 内部：確認 → 切替 → タイマー起動 → 応答。失敗なら `7F`。
:::
:::key
en: The mode change takes effect **before** the positive response leaves. From your next request on, the new session's services are live.
jp: モード変更は肯定応答が出る**前**に有効化されます。次のリクエストから新セッションのサービスが有効です。
:::

## leg:bytes
:::en
The positive response is **six bytes**: the echoed service, the session, and two timing numbers.
:::
:::jp
肯定応答は**6バイト**です ― 反響したサービス、セッション、そして2つのタイミング値。
:::
:::bytes
50 | SID+0x40 | pos
03 | session | sub
00 | P2 hi
32 | P2 lo
01 | P2* hi
F4 | P2* lo
:::
:::warn
en: These are **hex**. `00 32` = `0x0032` = 50 ms, not "32". `01 F4` = 500 × 10 ms = 5000 ms. Convert before reading.
jp: **16進**です。`00 32`＝`0x0032`＝50 ms（「32」ではない）。`01 F4`＝500×10 ms＝5000 ms。読む前に変換を。
:::

## leg:values
:::en
The session byte picks how much the ECU opens up.
:::
:::jp
セッションバイトが、ECUをどこまで開くかを選びます。
:::

## leg:values/card:v01
:::en
**Default** — normal running; only basic reads. The everyday state the ECU boots into.
:::
:::jp
**デフォルト** ― 通常走行。基本的な読み取りのみ。ECUが起動する日常の状態。
:::

## leg:values/card:v02
:::en
**Programming** hands over to the **bootloader** — the small resident program that can erase and rewrite the app flash. This is the door to reflashing, so `10 02` always starts a flash (Module 6).
:::
:::jp
**プログラミング**は**ブートローダ**へ引き渡します ― アプリのフラッシュを消去・書換できる常駐プログラム。再書込みの入口で、`10 02` は必ず再書込み（M6）の始まりです。
:::

## leg:values/card:v03
:::en
**Extended** unlocks deeper diagnostics: routines, actuator tests, and more data identifiers.
:::
:::jp
**拡張**はより深い診断を解放します：ルーチン、アクチュエータ試験、そして追加のデータ識別子。
:::
