$10 Diagnostic Session Control$10 診断セッション制御

The service that moves the ECU between its modes — from everyday Default into Extended or Programming, where deeper work is allowed.

ECUをモード間で動かすサービス ― 通常のデフォルトから、より深い作業が許される拡張・プログラミングへ。

Start here · まずここから

Before any bytes: a session is the ECU's operating mode. Ask for the right one and protected functions open up; ask at the wrong time and the ECU refuses. This section is the request that makes that switch.

バイトの前に：セッションとはECUの動作モードです。正しいものを頼めば保護された機能が開き、間違ったときに頼めばECUは断ります。この節は、その切り替えを起こすリクエストです。

CardC19

Requesting a session — and what the ECU does inside

セッションを要求する ― そのときECU内部で起きること
$10

High level · 概要

To change what an ECU will let you do, you ask it to switch session. The tester sends 10 <session>; the ECU checks the request, flips its internal mode, then replies 50 <session> … with the timing limits for that mode. One request, one response — but a real state change happens inside in between.

ECUに「できること」を変えてもらうには、セッションの切り替えを頼みます。テスターが 10 <session> を送ると、ECUは要求を確認し、内部モードを切り替え、そのモードのタイミング上限とともに 50 <session> … を返します。往復は1回でも、その間に本当の状態変化が起きています。

Go deeper (optional · 詳しく)

Inside the ECU — the switch itselfECU内部 ― 切り替えの実際▸

The switch isn't instant magic — the DCM (the ECU's diagnostic handler) runs a short internal sequence before it answers, and can still refuse.

DCM（ECUの診断担当）は、応答する前に短い内部手順を踏み、拒否することもあります。

Key · 要点The mode change takes effect before the positive response leaves. From your next request on, the new session's services are live.モード変更は肯定応答が出る前に有効化されます。次のリクエストから新セッションのサービスが有効です。

The session values — what each one unlocksセッション値 ― それぞれが開くもの▸

The session byte picks how much the ECU opens up.

セッションバイトが、ECUをどこまで開くかを選びます。

V0101 · Default
01・デフォルト

Default — normal running; only basic reads. The everyday state the ECU boots into.

デフォルト ― 通常走行。基本的な読み取りのみ。ECUが起動する日常の状態。

V0202 · Programming → bootloader
02・プログラミング → ブートローダ

Programming hands over to the bootloader — the small resident program that can erase and rewrite the app flash. This is the door to reflashing, so 10 02 always starts a flash (Module 6).

プログラミングはブートローダへ引き渡します ― アプリのフラッシュを消去・書換できる常駐プログラム。再書込みの入口で、10 02 は必ず再書込み（M6）の始まりです。

V0303 · Extended
03・拡張

Extended unlocks deeper diagnostics: routines, actuator tests, and more data identifiers.

拡張はより深い診断を解放します：ルーチン、アクチュエータ試験、そして追加のデータ識別子。

Section recap · まとめ

$10 moves the ECU between modes and reports the timing limits for the new mode; internally the DCM validates, flips the session, and arms its timers before it replies. You can now read a session-control exchange byte by byte. Next: how security ($27) gates the deeper sessions.

$10 はECUをモード間で動かし、新しいモードのタイミング上限を返します。内部ではDCMが確認し、セッションを切り替え、タイマーを起動してから応答します。これでセッション制御のやり取りをバイト単位で読めます。 次は、深いセッションをセキュリティ（$27）がどう守るか。
